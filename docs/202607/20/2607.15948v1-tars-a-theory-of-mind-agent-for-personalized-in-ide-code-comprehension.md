## 摘要
代码理解是软件工程中最耗时的任务之一，然而大多数基于大语言模型的助手生成的解释忽略了提问者的身份，并迫使开发者采用破坏性的复制粘贴工作流程。我们提出了TARS，一个集成在Visual Studio Code中的大语言模型驱动的代理，通过直接锚定于所分析代码的自主解释来支持程序理解。围绕轻量级的心智理论范式构建，TARS分析开发者的专业知识、角色和风格偏好，然后相应地调整其解释的深度和语气，并通过检索增强生成将其建立在项目文档的基础上。为评估TARS，我们对18名参与者进行了一项受控实验，实验中使用非平凡的Java代码片段。使用TARS的参与者完成任务的速度快26%，报告的认知负荷更低，并发现解释有有意义地适应了他们的个人特征。

## Abstract
Code comprehension is one of the most time-consuming tasks in software engineering, yet most LLM-based assistants produce explanations that ignore who is asking and force developers into a disruptive copy-paste workflow. We present TARS, an LLM-powered agent integrated into Visual Studio Code that supports program comprehension through autonomous explanations anchored directly to the code under analysis. Built around a lightweight Theory of Mind paradigm, TARS profiles a developer's expertise, role, and stylistic preferences, then adapts the depth and tone of its explanations accordingly, grounding them in project documentation via Retrieval-Augmented Generation. To evaluate TARS, we conducted a controlled experiment with 18 participants on non-trivial Java snippets. Participants using TARS completed tasks 26\% faster, reported lower cognitive load, and found the explanations meaningfully adapted to their profiles.

## 2. 方法论：核心思想与关键技术细节
-   **核心思想**：TARS 是一个集成在 VS Code 中的轻量级心智代理，它将“心智理论”范式显式化，通过预先建模的开发者画像来调整解释的深度、语气和粒度，并利用检索增强生成将解释扎根于项目文档。
-   **关键技术流程**：

    -   **有状态代理图**：基于 LangGraph 构建，包含三个核心节点：
        -   **规划节点**：接收代码片段、用户画像和检索到的相关文档，将所有指令注入一个统一的系统提示词中，要求大语言模型生成一个包含原始代码和自然语言解释的结构化 JSON 对象。这种将画像作为静态指令前置的方式，避免了运行时多步推理，优化了延迟。

## 6. 主要结论与发现
-   **性能加速而非纠错**：TARS 使用者的任务完成时间平均缩短了 26%，且完成时间的变异性显著降低，但答案的正确性与手动组几乎持平。这表明 TARS 的核心价值在于帮助开发者更快地建立起有效理解，而非直接提升理解的极限准确度。
-   **显著优化开发者体验**：问卷结果具有统计学显著性，参与者认为 TARS 在感知有用性和易用性上得分极高，同时报告了显著更低的心理需求、努力程度和挫败感，表明其有效降低了认知摩擦。
-   **个性化感知初步达成**：参与者普遍认为 TARS 的解释适应了他们的个人特征，验证了在编码助手中集成心智理论的可行性，但部分用户也反馈生成的详细度与自身偏好仍有差距。

## 7. 优点
-   **创新性的方法论融合**：首次将心智理论显式、前置地建模并集成到 IDE 代理中，为解决 LLM 工具“以一代全”的问题提供了新范式。
-   **以开发者为中心的体验设计**：解释直接锚定在编辑器内，并结合 RAG 落地于项目上下文，构建了一个流畅、低干扰的认知工作流。
-   **严谨的代理架构**：通过规划、语法检查、评判的循环代理设计，在低模型下提供了自我纠错能力，提升了解释输出的结构化与可靠性。
-   **多维度评估**：结合了客观任务指标、标准化认知负荷量表和技术接受模型，对工具的效用进行了立体验证。

## 8. 不足与局限
-   **外部效度不足**：实验仅在小规模（18 人）、实验室环境、单一语言（Java）和孤立代码片段上进行，结论能否推广至真实代码库和职业开发者群体仍待验证。
-   **统计显著性未达阈值**：虽然时间效率提升明显，但因样本量限制，其核心优势未在统计学上得到严格确认。

-   **缺少组件消融与横向对比**：未剥离评估 RAG、SoM 画像等关键组件的独立价值，也未与同类的 IDE 代码解释插件进行横向比较，难以准确归因优势来源。
-   **模型单一**：所有实验均基于 GPT-4.1 Nano，工具的效能对不同模型（尤其是轻量级开源模型）的鲁棒性未知。
