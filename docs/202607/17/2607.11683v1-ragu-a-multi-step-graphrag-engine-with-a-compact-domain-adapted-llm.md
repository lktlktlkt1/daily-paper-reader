## 摘要
图检索增强生成（GraphRAG）利用结构化知识增强大语言模型，但现有系统在单次抽取中构建知识图谱，产生有噪声的实体和脆弱的检索。RAGU是一个开源模块化GraphRAG引擎，通过将抽取与整合分离来解决这个问题：实体和关系经过两阶段类型化抽取、基于DBSCAN的去重、LLM摘要和Leiden社区检测。一个关键的洞见推动我们使用紧凑的抽取器：管道内LLM所需的技能——理解、抽取、基于上下文的推理——是语言技能，这些技能随模型规模增长而增长缓慢，不同于事实性世界知识。因此，我们训练了Meno-Lite-0.1，一个针对语言技能优化的7B模型，它在知识图谱构建上优于Qwen2.5-32B（相对调和均值+12.5%），并在英文GraphRAG任务上与之持平。在GraphRAG-Bench（医疗）上，RAGU在每个事实层面检索到最完整的上下文（证据召回率高达0.84 vs. ≤0.76），并在综合任务上超过了HippoRAG2；在多跳事实问答中，HippoRAG2的表面优势被证明主要是答案格式的伪影。RAGU可通过pip install graph_ragu安装，在单个GPU上运行，并以MIT许可证发布。源代码在https://github.com/RaguTeam/RAGU公开，Meno-Lite-0.1模型可从https://huggingface.co/bond005/meno-lite-0.1获取。

## Abstract
Graph retrieval-augmented generation (GraphRAG) enhances large language models with structured knowledge, yet existing systems construct knowledge graphs in a single extraction pass, producing noisy entities and brittle retrieval. RAGU, an open-source modular GraphRAG engine, addresses this by separating extraction from consolidation: entities and relations pass through two-stage typed extraction, DBSCAN-backed deduplication, LLM summarization, and Leiden community detection. A key insight motivates a compact extractor: the skills an in-pipeline LLM needs - comprehension, extraction, reasoning over context - are language skills that grow only weakly with model size, unlike factual world knowledge. Accordingly, we train Meno-Lite-0.1, a 7B model optimized for language skills, which outperforms Qwen2.5-32B on knowledge-graph construction (+12.5% relative harmonic mean) and matches it on English GraphRAG tasks. On GraphRAG-Bench (Medical), RAGU retrieves the most complete context at every factoid level (evidence recall up to 0.84 vs. $\leq$0.76) and overtakes HippoRAG2 on synthesis tasks; on multi-hop factoid QA, the apparent HippoRAG2 advantage is shown to be largely an answer-format artifact. RAGU is installable via $\texttt{pip install graph_ragu}$, runs on a single GPU, and is released under MIT. The source code is publicly available at https://github.com/RaguTeam/RAGU, and the Meno-Lite-0.1 model can be obtained from https://huggingface.co/bond005/meno-lite-0.1.

## 2. 论文提出的方法论

*   **核心思想**：GraphRAG管道应显式分离“抽取”与“合并”（整合）阶段，通过多步后处理消除单次抽取带来的噪声，而非直接使用大规模模型。
*   **RAGU系统的六步管道**：
    1.  **文本分块**：支持固定大小重叠分块、语义分块和智能语义分块。
    2.  **两阶段类型化抽取**：先抽取实体并基于NEREL模式（29种实体类型）进行校验，再在受约束的实体集内抽取关系。这种分离避免了实体-关系不匹配。
    3.  **合并去重与摘要**：对重复提及的实体和关系，分别应用DBSCAN聚类和LLM摘要，以降低图谱噪声。
    4.  **社区检测**：使用层次Leiden聚类对去重后的图谱进行划分。
    5.  **社区摘要**：由LLM为每个社区生成结构化报告（标题、摘要、要点）。
    6.  **图谱细化**：可通过可插拔模块（如移除孤立节点）进行优化。
*   **搜索引擎**：提供了本地搜索（向量相似度）、全局搜索（社区摘要打分）、朴素搜索（标准向量RAG）、混合搜索和查询计划分解（DAG）五种引擎。
*   **紧凑模型 Meno-Lite-0.1**：
    *   **基座**：从RuadaptQwen2.5-7B出发。
    *   **训练**：先进行13亿token的持续预训练（俄语+英语教育/科学文本），再进行5000万token的有监督微调（覆盖NEREL抽取、多跳问答等），旨在强化模型利用上下文而非回忆事实的能力。
    *   **特性**：支持128K上下文窗口，俄语分词效率提升47%，可通过vLLM在单张消费级GPU上部署。

*   **语言技能假说成立**：Meno-Lite-0.1（7B）在知识图谱构建任务上的调和均值比Qwen2.5-32B高出12.5%，尤其体现在更依赖语言理解的关系抽取上，证明了管道内LLM的能力与模型规模增长弱相关。
*   **RAGU与HippoRAG2的优势互补**：
    *   **RAGU**擅长需要综合广泛上下文的生成任务（如医学领域的创造性生成、上下文摘要），在所有任务上均能检索到最完整的证据（证据召回率最高达0.84）。

*   **紧凑模型可作为API替代品**：在RAGU框架内，使用本地Meno-Lite-0.1与使用API大模型（gpt-oss-20b）的结果差距极小，实现了在单GPU预算下接近昂贵API的性能。

## 8. 不足与局限

*   **实验覆盖范围**：缩放定律的证据主要来自Qwen2.5单一模型家族和特定任务（CheGeKa, MultiQ），虽具有启发性，但无法作为普遍定理。
*   **潜在偏差风险**：
    *   模型Meno-Lite-0.1的有监督微调用了NEREL数据集的训练/验证集，而知识图谱构建基准用了NEREL的留存测试集。虽然文档不重叠，但共享的标注模式和文本域可能带来残余优势。
    *   NEREL模式本身源于俄语新闻文本，应用于其他语言或领域可能需适配。
*   **应用限制**：
    *   Meno-Lite-0.1以牺牲参数化事实记忆为代价换取上下文处理能力，不应被用作独立知识库；其在超过32K令牌时的多跳推理能力会下降。
    *   RAGU的默认NetworkX图后端不适用于百万级节点的大规模语料库，此时需切换到专用图数据库适配器。
    *   最终的图谱质量仍对底层的抽取LLM实力（特别是其基础能力）敏感，弱基座模型引入的结构噪声无法被后续合并完全修复。
