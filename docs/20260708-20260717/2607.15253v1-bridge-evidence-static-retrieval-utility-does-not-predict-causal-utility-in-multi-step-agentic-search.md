## 摘要
检索系统基于静态有用性观念进行训练和评估：向阅读器模型提供一篇文章和一个问题，观察答案是否改善，并据此对文章评分。当单独阅读一篇文章时，这一观念成立。但当语言模型作为搜索智能体时，它会发出多个查询并在多个回合间进行推理，此时文章的重要性可能在于它让智能体下一步能做什么，而不是它关于当前问题说了什么。我们测量了这一差距而非仅仅论证它。使用 HotpotQA 上的 ReAct 风格智能体，我们重放了 1000 个开发问题，并对智能体阅读的每一篇文章，将其删除并从该点重新运行轨迹的其余部分。将原始运行与反事实运行进行比较，得到一个反事实轨迹效用（CTU）分数，该分数来自三个变化量：最终答案质量、下一个查询的检索质量以及回合数。将 CTU 与基于 23322 个文档观察的静态 RAG 效用（SRU）进行交叉比较，两者几乎统计独立（斯皮尔曼秩相关系数 ρ = -0.026）。智能体阅读的文档中约有三分之一在因果上承担着重要角色，但静态阅读器却认为它们无用；我们称这些文档为桥梁文档。当用 BM25 和交叉编码器代理替换基于阅读器的轴时，该模式仍然存在，在均匀分布的轴上给出了 27.2% 的桥梁单元。第二个实验确定了机制。使用先前工作中的可观察实体相关性（OER）度量，在智能体的下一个查询中，区分相关候选与不相关候选的实体出现的频率是仅在不相关文档中发现的实体的 4.02 倍（6.1% 对 1.5%，n = 227,139）。桥梁文档通过为智能体提供一个具有区分性的实体来改变搜索方向，从而发挥作用。静态相关性与因果有用性在智能体检索中是不同的量，优化前者并不能带来后者。

## Abstract
Retrieval systems are trained and evaluated on a static idea of usefulness: hand a document and a question to a reader model, see whether the answer improves, and score the document accordingly. The idea holds up when a document is read on its own. It breaks when a language model works as a search agent, issuing several queries and reasoning across turns, because a document can matter for what it lets the agent do next rather than for what it says about the current question.   We measure that gap rather than argue it. Using a ReAct style agent over HotpotQA, we replay 1000 development questions and, for every document the agent read, delete it and re-run the rest of the trajectory from that point. Comparing the original run against its counterfactual gives a Counterfactual Trajectory Utility (CTU) score from three deltas: final answer quality, next query retrieval quality, and turn count. Crossing CTU against Static RAG Utility (SRU) over 23,322 document observations, the two are close to statistically independent (Spearman rho = -0.026). Roughly a third of the documents the agent reads are causally load bearing while looking useless to a static reader; we call these bridge documents. The pattern survives when the reader based axis is swapped for a BM25 and cross encoder proxy, giving a bridge cell of 27.2% on an evenly spread axis.   A second experiment pins down the mechanism. Using the Observable Entity Relevance (OER) measure from prior work, entities that discriminate relevant from non-relevant candidates appear in the agent's next query 4.02 times more often than entities found only in non-relevant documents (6.1% vs 1.5%, n = 227,139). A bridge document earns its keep by handing the agent a discriminative entity that redirects the search. Static relevance and causal usefulness are different quantities in agentic retrieval, and optimizing the first does not deliver the second.

### 1. 论文的核心问题与整体含义
- **研究背景**：当前检索系统（包括 RAG 评估）依循静态有用性范式：将文档和问题交给一个无历史的阅读器，观察答案是否提升，并据此打分。
- **问题所在**：在智能体式多步搜索中（如 ReAct 代理），文档的价值不再是直接回答当前问题，而可能是为智能体提供下一步搜索所需的线索（例如一个实体），从而间接促成任务成功。这类“桥梁文档”本身可能不包含最终答案，静态指标会将其判为无用，但它们对轨迹而言具有因果必要性。
- **整体含义**：论文的核心洞见是 **静态检索效用与因果效用是两个近乎独立的量**；当前检索系统所优化的信号无法捕捉智能体真正需要的文档，因此评估体系和训练目标需要从静态相关性转向因果有效性。

### 2. 论文提出的方法论
- **核心思想**：通过 **反事实轨迹探索**（Counterfactual Trajectory Exploration, CTE）量化单个文档对智能体轨迹的因果贡献。
- **关键技术细节**：
  - 以 ReAct 风格智能体（基于 Qwen2.5-7B-Instruct）在 HotpotQA 上运行多步问答，记录每一步阅读的文档。
  - **遗漏干预**：对智能体实际阅读的每一个文档，在第 \(t\) 步将其从候选列表中删除，然后以该点为起点重新执行（固定前缀），生成一条“反事实轨迹”。
  - **反事实轨迹效用（CTU）**：比较原轨迹与反事实轨迹，综合三个变化量：
    - \( \Delta \text{answer} = \text{F1}_{\text{base}} - \text{F1}_{\text{cf}} \)（答案质量下降多少）
    - \( \Delta \text{next-query} = \text{nDCG@10}(q^{\text{base}}_{t+1}) - \text{nDCG@10}(q^{\text{cf}}_{t+1}) \)（下一步检索质量下降多少）
    - \( \Delta \text{effort} = \text{turns}_{\text{cf}} - \text{turns}_{\text{base}} \)（所需回合数是否增加）
  - 对每个分项 min-max 归一化后等权加和得到 CTU。当 CTU = 1.4 时为无因果效应点。
  - **静态 RAG 效用（SRU）**：将文档和问题交给无历史的阅读器，计算答案 F1 相对于零样本（无文档）时的提升。SRU 高表示静态有用。
- **象限分析**：以 SRU > 0 和 CTU > 1.4 为阈值，将文档分为四类：预期有用（A）、冗余证据（B）、**桥梁证据（C）**、真正无用（D）。重点观察 C 格比例。
- **实体传播机制验证**：
  - 使用 **可观察实体相关性（OER）** 度量，将每一步候选集中的实体分为高 OER（能够区分相关/不相关文档）和低 OER（仅出现在不相关文档中）。
  - 统计这些实体在智能体下一步查询中出现的频率（精确匹配或部分匹配），检验假设：高 OER 实体被传播的比率远高于低 OER 实体。

### 6. 论文的主要结论与发现
- **CTU 与 SRU 几乎独立**：斯皮尔曼秩相关系数 ρ = -0.026，几乎无线性相关。
- **桥梁文档占据显著比例**：约 35.7% 的文档属于 C 格（低 SRU、高 CTU），即静态无用但因果必需。
- **替换静态轴后现象稳健**：以 BM25+交叉编码器为代理时，桥梁格仍占 27.2%，且与 CTU 的秩相关仍接近于零。这说明即使是部署系统中常用的检索信号也无法捕获因果效用。
- **实体传播是机制**：高 OER 实体进入下一步查询的概率是低 OER 实体的 4.02 倍（精确匹配）。桥梁文档正是通过提供这些可区分实体来重定向搜索，从而发挥因果作用。
- **静态相关性与因果有用性在智能体检索中是本质不同的量**，优化前者无法获得后者。

### 7. 优点
- **因果干预设计严谨**：使用遗漏单个文档并固定轨迹前缀的反事实方法，直接测量因果效应，而非依赖事后相关性。
- **多维度 CTU 指标**：联合考虑答案质量、下一步检索效率和回合数变化，并发现其零效应点与中位数自然重合，为阈值选择提供了理论依据。
- **稳健性验证**：不仅在基于阅读器的 SRU 上验证，还改用检索系统原生的 BM25 与交叉编码器代理，增强了结论的普适性。
- **机理解释**：从实体传播角度给出桥梁文档为何有效的具体解释，并用 OER 度量进行了定量验证，连接了静态实体信号与智能体动态行为。
- **诚实对待局限**：主动暴露并分析了静态轴偏斜、轮次分组混淆、实体定位假设未通过等弱点，提升了科学可信度。

### 8. 不足与局限
- **静态轴极度偏斜**：ΔSRU > 0 的样本仅占 3.3%，导致桥梁格比例主要由边缘分布和独立性预测而来，独立性的发现比具体百分比更重要。
- **静态阅读器能力弱**：多跳问题的单文档答案 F1 极低，可能削弱 SRU 作为“静态有用性度量”的效力。不过 BM25 代理实验部分缓解了此担忧。
- **代理象限排除大量数据**：BM25 与交叉编码器信号一致的文档仅占约 41%，被排除的数据可能行为不同，结论外推性受限。
- **CTU 依赖于特定智能体**：所有测量基于 Qwen2.5-7B 这一个模型和特定提示，结论是否迁移至其他模型或更复杂系统尚未验证。
