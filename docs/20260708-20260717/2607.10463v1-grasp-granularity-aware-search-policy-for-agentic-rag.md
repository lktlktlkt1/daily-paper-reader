## 摘要
代理式检索增强生成（RAG）通过允许语言模型迭代推理、生成搜索查询、检索证据并预测答案，扩展了静态 RAG。然而，模型仍然难以决定何时检索、使用词汇匹配还是语义相似性，以及如何控制上下文粒度，以防止无关信息干扰代理的推理。在本文中，我们提出了 GRASP，一个强化学习（RL）框架，用于训练代理在多步推理过程中自适应地协调互补的检索工具。GRASP 为代理提供语义搜索、关键词搜索和段落阅读动作，使其能够检索句子级证据，并仅在需要时才扩展更多上下文。我们使用同时考虑答案准确性、有依据的阅读、互补搜索和轮次效率的奖励来训练策略。在多跳推理基准上的实验表明，与单步检索、基于提示的代理式 RAG 和基于 RL 的检索基线相比，GRASP 提高了检索召回率和下游问答性能。定性和消融分析表明，学习到的策略形成了可解释的略读和扫描行为：它使用语义搜索进行广泛探索，使用段落阅读进行局部验证，使用关键词搜索获取实体特定证据。这些结果表明，学会协调检索信号和上下文粒度对于代理的正确推理至关重要。

## Abstract
Agentic retrieval-augmented generation (RAG) extends static RAG by allowing language models to iteratively reason, generate search queries, retrieve evidence, and predict answers. However, it remains challenging for models to decide when to retrieve, whether to use lexical matching or semantic similarity, and how to control context granularity to prevent irrelevant tokens from interfering with agent reasoning. In this paper, we introduce GRASP, a reinforcement learning (RL) framework for training agents to adaptively coordinate complementary retrieval tools during multi-step reasoning. GRASP provides the agent with semantic search, keyword search, and paragraph-reading actions, enabling it to retrieve sentence-level evidence and expand further context only when needed. We train the policy with a reward that jointly accounts for answer accuracy, grounded reading, complementary search, and turn efficiency. Experiments on multi-hop reasoning benchmarks show that GRASP improves both retrieval recall and downstream question answering performance compared with single-step retrieval, prompting-based agentic RAG, and RL-based retrieval baselines. Qualitative and ablation analyses show that the learned policy develops interpretable skimming and scanning behavior: it uses semantic search for broad exploration, paragraph reading for local verification, and keyword search for entity-specific evidence. These results suggest that learning to coordinate retrieval signals and context granularity is critical for agent's correct reasoning.

*   **整体含义**：论文旨在证明，通过强化学习训练一个能自适应协调多种检索工具和上下文粒度的代理，是解决上述挑战、提升复杂推理任务性能的关键。

### 2. 论文提出的方法论

*   **核心思想**：将代理式RAG建模为一个有限视界的马尔可夫决策过程，并引入强化学习框架GRASP，训练语言模型代理在多步推理中动态、自适应地协调使用互补的检索工具和上下文扩展动作。
*   **关键技术细节**：
    *   **动作空间设计**：
        1.  `τs` (语义搜索): 基于语义相似性，从句子级语料库中检索与查询相关的Top-K句子。
        2.  `τk` (关键词搜索): 基于词汇匹配（BM25），从句子级语料库中检索与查询精确或接近匹配的Top-K句子。
        3.  `τr` (阅读段落): 根据已检索到的句子ID，返回其所在的完整父段落，以提供更丰富、粒度的上下文。
        4.  `τa`: 终止搜索并生成最终答案。
    *   **奖励函数设计**：轨迹级的总奖励R由四个部分组成，以引导智能体的行为：
        *   $R = R_A + \alpha R_R + \beta R_S + \gamma R_E$
        *   **答案准确性 ($R_A$)**：计算预测答案与标准答案的Token级F1分数。
        *   **有依据的阅读 ($R_R$)**：评估代理调用“阅读段落”动作时，是否选中了包含黄金证据的段落，鼓励阅读有效证据并惩罚阅读干扰项。
        *   **互补搜索 ($R_S$)**：当且仅当语义搜索和关键词搜索都至少检索到一篇黄金文档时，才给予奖励。这鼓励代理利用互补的检索信号。
        *   **轮次效率 ($R_E$)**：在答案正确的前提下，奖励使用更少轮次（行动步数）的轨迹，以鼓励效率。
        *   **权重分配**：$\alpha = 0.7$, $\beta = 0.15$, $\gamma = 0.15$。权重设计将主要辅助信号放在有依据的阅读上，同时平衡搜索质量和效率，并确保答案准确性仍是驱动策略学习的主要因素。
    *   **学习算法**：采用**组相对策略优化（GRPO）** 算法。对于每个问题，策略生成一组G个轨迹，然后根据组内这些轨迹的相对奖励来更新策略，而非依赖绝对奖励值。
    *   **实现细节**：仅对模型自生成的Token（如推理、工具调用、最终答案）计算策略损失，对检索工具返回的观测文本进行掩码处理。

### 6. 论文的主要结论与发现

*   **综合性能最优**：GRASP在三个多跳推理基准上一致地取得了最佳的检索召回率和问答性能，显著优于单步检索、基于提示和基于RL的代理式RAG基线方法。
*   **检索行为可解释且高效**：学习出的策略展现出类似人类信息搜寻的行为模式，即“略读”和“扫描”：
    *   **语义搜索** 用于广泛的、概念性的探索。
    *   **段落阅读** 用于对候选证据进行上下文验证，提取“桥接实体”。
    *   **关键词搜索** 转向针对特定实体的精准词汇检索。
    *   这种模式使代理能够有效验证信息，避免过早做出错误的推理承诺。
*   **各组件均有关键作用**：消融实验证实，互补的搜索信号（语义和关键词）以及可调节的上下文粒度（通过段落阅读动作实现）都是贡献最终性能的关键设计。
    *   移除语义搜索对性能的损害比移除关键词搜索更大
    *   移除段落阅读（即固定使用段落粒度）导致性能下降最严重，因为代理失去了精细化控制上下文和验证信息的能力。

### 7. 优点

*   **问题定义新颖且重要**：首次将“检索信号选择”、“上下文粒度控制”和“多步检索策略学习”三个关键挑战联合在一个统一的强化学习框架下解决。
*   **方法论设计精巧**：
    *   **动作空间** 设计直观且强大，模拟了人类信息检索的逻辑步骤。
    *   **奖励函数** 设计细致，通过多目标函数精确引导模型形成理想检索行为，并有效防止奖励黑客行为。
*   **分析深入透彻**：不仅提供了性能对比，还通过马尔科夫链分析和案例展示深入剖析了模型学到的行为模式，为理解模型决策提供了可解释性。消融实验严谨地验证了各组件的价值。
*   **实验设置公平全面**：对比基线覆盖了多种范式，评估指标涵盖了检索和生成两方面，并评估了跨数据集泛化能力。

### 8. 不足与局限

*   **对黄金标注的依赖**：奖励函数中的“有依据的阅读”和“互补搜索”奖励严重依赖问答对中的黄金支持事实标注，限制了该方法在无标注语料库上的广泛应用。
*   **策略仍不完美**：即使在训练后，代理有时仍会在未检索到充分证据时过早终止并给出答案，导致推理失败。
*   **模型规模单一**：所有实验仅基于一个3B参数的基座模型进行。不同的模型规模可能会展现出不同的策略学习动态，论文未对此进行分析。
*   **计算成本高**：训练多步检索代理强化学习策略的计算成本较高，这限制了大规模网格搜索式超参数优化的可行性。例如，消融实验不得不在数据子集上进行。
