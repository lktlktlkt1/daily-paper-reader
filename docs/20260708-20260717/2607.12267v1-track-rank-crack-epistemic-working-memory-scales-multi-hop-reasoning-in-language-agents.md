## 摘要
将推理与工具调用交织进行的语言代理，随着推理链的加长，性能急剧下降，即使每个单独步骤都很简单。我们将此归因于上下文稀释：代理的探查状态（已确认的内容、怀疑的内容、仍需查明的内容）仅隐含在不断增长的上下文窗口中，早期的发现被后续的检索所淹没。我们提出 SLEUTH，它通过结构化的认知工作记忆使这一状态显式化且可操作：代理维护基于来源的已确认事实、按证据排序的活跃假设，以及直接驱动下一步行动的开放问题。在五个多跳基准和五个已有基线模型上，SLEUTH 的优势随着难度增加而扩大，从 HotpotQA 上的 +5 分到四跳链上的 +11 分，甚至无需多轮迭代就超越了 Reflexion。分析剩余差距所在，我们发现了证据充分性问题：代理经常找到了答案却未能最终确认，将预算浪费在多余的验证上。一个轻量级的确认触发器解决了这一问题，但前提是代理已经维护了结构化状态：相同的触发器应用于非结构化代理时毫无改进，表明有组织的认知状态是有效确认的必要条件。最后，在较弱模型上强制遵循协议可在最难问题上恢复多达 +19 分，表明代理如何组织其推理，而非原始模型能力，才是扩展多跳推理的关键因素。

## Abstract
Language agents that interleave reasoning and tool use degrade sharply as reasoning chains lengthen, even when each individual step is easy. We trace this to context dilution: an agent's investigative state (what it has confirmed, what it suspects, and what it still needs) lives only implicitly in a growing context window, where early discoveries are buried under later retrievals. We introduce SLEUTH, which makes this state explicit and actionable through a structured epistemic working memory: the agent maintains Confirmed Facts grounded to sources, Active Hypotheses ranked by evidence, and Open Questions that directly drive its next action. Across five multi-hop benchmarks and five established baselines, SLEUTH's advantage grows with difficulty, from +5 points on HotpotQA to +11 on 4-hop chains, surpassing Reflexion without multiple episodes. Analyzing where the remaining gap lies, we identify the evidence sufficiency problem: agents often find the answer but fail to commit, exhausting their budget on needless verification. A lightweight commitment trigger fixes this, but only when the agent already maintains structured state: the identical trigger applied to an unstructured agent yields no improvement, isolating organized epistemic state as the necessary condition for effective commitment. Finally, enforcing protocol adherence on a weaker model recovers up to +19 points on the hardest problems, showing that how an agent organizes its reasoning, not raw model capability, is the active ingredient for scaling multi-hop reasoning.

### 1. 论文的核心问题与整体含义

*   **核心问题**：语言代理在交叉进行推理和工具调用以解决多跳问题时，会遭遇**上下文稀释（Context Dilution）**。随着推理链加长，早期发现的关键信息会被后续大量的检索结果淹没在上下文窗口中，导致代理丢失探查状态，性能急剧下降。即使单个推理步骤很简单，长链的组成依然会带来巨大的困难。
*   **整体含义**：论文指出，在多跳推理中，瓶颈不在于模型的“推理能力”，而在于“推理的组织方式”。通过引入一种模仿人类调查员工作方式的结构化认知工作记忆，可以显式地管理“已知、怀疑、未知”状态，从而有效扩展语言代理的多跳推理能力。

### 2. 论文提出的方法论

*   **核心思想**：提出 **SLEUTH (Structured Layered Evidence-Updated Tracking of Hypotheses)** 框架，通过提示工程，让代理在每一步都维护并输出一个结构化的**认知工作记忆（Epistemic Working Memory）**，以此代替隐式存在于上下文中的状态。
*   **关键技术细节**：
    *   **工作记忆三元组**：代理在每一回合 `t` 都需要更新并输出状态 `Wt = (Ft, Ht, Qt)`。
        *   **已确认事实 (`Confirmed Facts Ft`)**：从工具输出中提取的、带有来源引用的已验证信息。采用单调性积累原则，一旦确认就不撤回，以防止证据丢失。
        *   **活跃假设 (`Active Hypotheses Ht`)**：候选答案的排序列表，每个假设都关联着支持或反对它的已确认事实，并有明确的置信度（高/中/低）。
        *   **开放问题 (`Open Questions Qt`)**：当前存在的、其解答能最大程度改变假设排名的未解决问题，每个问题都附带一个提议的搜索行动和优先级。
    *   **行动选择机制**：代理的下一步工具调用由最高优先级的开放问题直接驱动，实现了认知状态与行动的紧密耦合。
    *   **更新协议**：代理在每次工具调用后，依次执行：提取新事实、更新假设的置信度和排名、解决或生成开放问题、选择下一步行动。
    *   **确认触发器（Commitment Mechanism / Nudge）**：为解决“证据充分性问题”（代理已找到答案但因无止境验证而耗尽预算），在推理预算的70%处引入一个轻量级触发器。当回合数达标且事实数量足够时，强制代理基于现有结构化状态合成答案，实现从探索到利用的转换。

### 6. 论文的主要结论与发现

*   **结构化工作记忆优势随难度扩展**：SLEUTH在五基准上均取得最高分，且对ReAct的优势从2跳的约+11%扩展到4跳的+11.1%（EM），表明推理链越长，结构化状态越重要。
*   **定位“证据充分性问题”**：代理频繁在找到答案后因过度验证而超时。SLEUTH的提交触发器通过强制利用已组织的证据来解决此问题，但若无结构化状态，触发器本身无效。
*   **协议遵循是核心驱动因素**：在弱模型GLM-5上强制其遵循SLEUTH的结构化协议，可在最难（4跳）问题上恢复高达19个EM点。结论表明，“如何组织推理”是比“模型原始能力”更关键的扩展多跳推理的要素。

### 7. 优点

*   **方法简洁有效**：完全基于提示工程，无需微调、训练或额外基础设施，具有广泛适用性。
*   **可解释性强**：工作记忆以自然语言形式输出，每一步的认知状态、推理过程和行动依据都清晰透明，便于调试和审计。
*   **实验论证严谨**：消融实验设计精妙，成功隔离了不同组件和机制的贡献。特别是“提交触发器+非结构化代理”的对照实验，强有力地证明了结构化状态是有效确认的必要条件。
*   **洞察深刻**：区分了“信息丢失”和“信息瘫痪”两种失败模式，并将其与SLEUTH不同组件的功能精准对应。

### 8. 不足与局限

*   **单次推理限制**：工作记忆在单次任务内有效，无法跨任务积累经验（与Reflexion等不同）。
*   **事实单调性的风险**：已确认事实不可撤销的设计可能在早期提取到误导性信息时导致错误固化，论文承认了此问题但在当前封闭设定下影响有限，在开放域、多源冲突环境中风险更高。
*   **提交阈值的人工设置**：触发器的最佳时机（70%预算）是在4跳MuSiQue上调试得出的，迁移到探索/利用比例差异大的任务时可能需要重新调试或设计自适应机制。
*   **应用场景限制**：实验局限于封闭语料库的多跳问答任务，其在真实开放域或更多样化的代理任务中的表现有待考证。
