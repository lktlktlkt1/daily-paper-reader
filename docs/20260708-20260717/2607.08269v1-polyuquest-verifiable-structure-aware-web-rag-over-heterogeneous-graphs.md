## 摘要
现有的检索增强生成（RAG）系统将网页视为纯文本，丢失了HTML中编码的结构和语义信号。我们提出PolyUQuest，一个可验证的、结构感知的网络RAG框架，它构建在异构图之上，统一了页面间的超链接拓扑、页面内的DOM层次结构以及跨页面的实体关系知识。一个两级路由器根据查询的结构需求将其分派到三种检索模式之一，包括直接块检索、跨页面图遍历和多跳实体推理。每个答案都是完全可验证的，因为每个引用块都携带其源页面、标题路径和实体链接，用户可以追溯任何声明到其结构证据。我们在香港理工大学（PolyU）的官方网站上进行了评估，包含4,240个页面、31,086个DOM块、29,119个实体和37,680个关系，以及一个多类型评估基准。PolyUQuest在答案正确性、覆盖度和忠实性上优于现有RAG系统，同时每次查询消耗的大模型令牌数显著减少。演示提供了一个交互界面，用于检查引用答案、比较不同路由模式下的检索轨迹以及探索证据图路径。PolyUQuest正准备部署为香港理工大学面向学生的问答服务。

## Abstract
Existing retrieval-augmented generation (RAG) systems treat web pages as flat text, losing the structural and semantic signals encoded in HTML. We present PolyUQuest, a verifiable, structure-aware web RAG framework built on a heterogeneous graph that unifies hyperlink topology between pages, DOM hierarchy within pages, and entity-relation knowledge across pages. A two-tier router dispatches each query to one of three retrieval modes matched to its structural need, including direct block retrieval, cross-page graph traversal, and multi-hop entity reasoning. Every answer is fully verifiable, as each cited block carries its source page, heading path, and entity links so that users can trace any claim back to its structural evidence. We evaluate on the official websites of the Hong Kong Polytechnic University (PolyU), comprising 4,240 pages, 31,086 DOM blocks, 29,119 entities, and 37,680 relations, together with a multi-type evaluation benchmark. PolyUQuest outperforms existing RAG systems in answer correctness, coverage, and faithfulness, while consuming significantly fewer LLM tokens per query. The demonstration provides an interactive interface for inspecting cited answers, comparing retrieval traces across routing modes, and exploring evidence graph paths. PolyUQuest is being prepared for deployment as a student-facing QA service at PolyU.

## 1. 核心问题与整体含义

- **研究动机**：网页是RAG系统最常用的知识源之一，但现有方案普遍将网页视为扁平文本块，丢弃了HTML中原生的**结构信号**（如超链接拓扑、DOM层级、跨页实体关联）。当问题需要跨页综合信息（例如“哪位教授既做NLP又教相关课程？”）时，纯文本检索无法有效导航和聚合散落在多个页面中的证据。
- **核心缺口**：现有系统或只保留单页DOM结构却忽略超链接（如HtmlRAG），或仅利用实体图却仍将HTML展平并注入过大的全局上下文（如LightRAG），缺乏一种同时利用**站点拓扑、块级层次和跨页实体**的机制，且答案的**可验证性**普遍缺失。
- **整体含义**：PolyUQuest 提出了一个**可验证、结构感知**的网络RAG框架，通过构建统一的异构图，让系统能根据查询的结构需求灵活调度多模式检索，最终输出携带完整溯源（源页面、标题路径、实体链接）的答案，旨在解决大学等组织机构网站上的复杂信息问答。

## 2. 方法论：核心思想与技术细节

### 2.1 三层异构图的构建（离线阶段）
- **节点类型**：\(V = V_P \cup V_B \cup V_E \cup V_T\)，分别表示网页、证据块、实体和主题。
- **边类型**：捕获四种连接——页面间超链接、页面‑块包含、块‑实体提及、实体/主题间的语义关联。
  - **第一层（站点图）**：页面间的超链接。
  - **第二层（块树）**：基于清洁后DOM构建的标题感知文本块，保留页面内层级（合并兄弟节点至约150词阈值，保留表格/列表完整性，过滤导航栏等模板区域）。
  - **第三层（实体图）**：通过LLM从块中抽取实体和关系（人、项目、系、课程、研究方向等），经别名规范化与语义‑字符串相似度消歧后，将实体链接回源块。
- **可回溯设计**：每个证据块 \(b\) 记录其源页面 \(p(b)\)、标题路径 \(h(b)\)、向量嵌入 \(x_b\)、长度 \(\ell(b)\) 及包含的实体集 \(V_E(b)\)。

### 2.2 在线结构感知检索（在线阶段）
- **两级路由器**：
  - **第一级轻量规则**：拦截明确的模式词（如“which professors” → Mode C，“admission requirements for” → Mode B，对比词 → Mode B）。
  - **第二级LLM分类器**：处理未命中规则的查询，赋予三种模式之一的置信度。
- **三种检索模式**：
  - **Mode A – 直接块检索**：适用于单跳事实查询。先通过密集ANN结合BM25召回候选块集 \(B_q\)，再用交叉编码器针对查询、块文本、源页面和标题路径进行重排序，最后用父块上下文丰富后生成答案。
  - **Mode B – 导航检索**：适用于跨页聚合查询。先将查询分解为子查询，对每个子查询检索相关页面并沿超链接扩展邻居页面，收集这些页面上的证据块，进行整体重排序和合成。
  - **Mode C – 实体推理**：适用于多跳实体问题。先用LLM提取话题关键词，通过两条路径（实体‑实体关系扩展、话题‑实体关联扩展）定位相关实体节点 \(V_{E,q}\)，再回溯到源块。块评分采用公式：
    \[
    s(b,q) = \cos(x_q, x_b) + \frac{\min(|V_E(b) \cap V_{E,q}|, \kappa)}{\kappa} + \left(1 - \frac{\ell(b)}{\max_{b'}\ell(b')}\right)
    \]
    三项依次衡量查询‑块语义相似度、实体覆盖度（上限 \(\kappa=30\)）和块简洁性，同时引入同页惩罚促进来源多样性。
- **证据组装与答案生成**：重排序后的块附带源页面、标题路径等元数据，LLM据此生成带内联引文的答案，并在界面显示检索轨迹。

## 3. 实验设计

### 3.1 数据集与场景
- **数据规模**：爬取的香港理工大学官方英文页面 4,240 页，解析为 31,086 个 DOM 证据块，提取并消歧出 29,119 个实体、37,680 条关系。
- **评估基准**：构建了包含 300 个问题的 **PolyU-Web** 数据集，覆盖三种检索需求：单页事实查询、跨页导航查询、多页实体推理查询。对每个问题，人工标注了相关的官方页面和参考证据块。

### 3.2 对比方法
- **纯文本块RAG**：ChunkRAG（固定大小文本块）
- **HTML结构RAG**：HtmlRAG（保留DOM，但不利用超链接）
- **图RAG**：FastGraphRAG、LightRAG（基于实体图，但展平HTML并注入大量上下文）

## 6. 主要结论与发现

- PolyUQuest 在**正确性(0.644)、覆盖度(0.649)、忠实性(0.921)** 上全面超越所有基线，其中忠实性领先第二名的 HtmlRAG 达 36%。同时，每次查询令牌数（2,968）仅为 LightRAG 的 1/10，也比 HtmlRAG 低 26%，成本效益显著。
- DOM 感知的块划分是性能提升的主因：消融实验中替换为固定块后正确性和覆盖度大幅下降（-8.1和-13.9分）。
- 跨页导航模式的贡献相对较小，但在特定查询中保证了聚合能力。
- 路由机制避免了注入大段全局上下文，使低成本与高质量得以兼得。

## 7. 优点

- **统一的结构建模**：首次将页面链接、DOM层次和跨页实体知识融合在单一异构图中，突破了现有工作仅侧重单层结构的局限。
- **查询驱动的高效检索**：两级路由器与三种匹配模式的设计，精准提供结构所需的证据，避免了全局上下文的膨胀，大幅降低了LLM消耗。
- **全链路可验证性**：每个引用块携带源页面、标题路径、实体链接，并提供交互界面审查检索轨迹、证据块和实体图路径，提升了系统的可信度与可解释性。
- **现实场景部署**：以大学官网为实例，正在准备作为学生问答服务上线，论证了方法的实用价值。

## 8. 不足与局限

- **领域单一性**：实验仅基于香港理工大学一个机构网站，虽然讨论了一般化潜力，但缺少在政府、医院、企业等其他组织网站的验证，泛化性待证实。
- **实体模式依赖**：离线实体抽取需要预定义类型（人、项目等），迁移至新领域时需重新设计模式并可能调整消歧策略，自动化程度有限。
- **实验覆盖面有限**：缺少对路由准确率的独立评估、不同块大小敏感度测试、消融实体消歧贡献等更细粒度的分析；也未对比其他保持结构的 RAG 变体（如图结构保留方法）。
- **动态性与实时性未评估**：论文未讨论网页更新的增量索引或路由延迟，对于实际部署中的响应时间和维护成本缺乏量化。
- **语言限制**：只处理英文页面，未涉及多语言或包含复杂媒体（图片、表格深度理解）的场景。实体推理依赖 LLM 抽取和消歧，可能引入级联错误。
- **可扩展性**：异构图的规模随网站增大而膨胀，30k+实体和块的索引能否在更大规模站点保持低延迟未作讨论。
